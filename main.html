GIT

A. Pengertian

  • Menurut Linus Torvalds (Pencipta Git):

    • Git adalah sistem kontrol versi yang sangat cepat dan sangat efisien dengan dukungan untuk distribusi non-linear penuh (ribuan cabang bersamaan dengan ratusan ribu pengecekan) yang sangat cocok untuk proyek besar seperti kernel Linux.

  • Menurut Scott Chacon (Penulis buku "Pro Git"):

    • Git adalah alat manajemen kode sumber yang dirancang untuk sangat cepat dan sangat efisien. Ini adalah sistem kontrol versi yang mendistribusikan, yang berarti bahwa alih-alih memiliki satu salinan kode sumber dan riwayatnya di satu tempat, setiap anggota tim memiliki salinan lengkap.

B. Fungsi Git

  1. Melacak perubahan dalam kode sumber:
    2. Git menyimpan riwayat lengkap dari setiap perubahan yang dibuat dalam kode proyek, memungkinkan pengembang untuk mengidentifikasi perubahan spesifik dan kapan perubahan tersebut dilakukan.

  2. Kolaborasi tim:
    3. Git memungkinkan tim pengembang untuk bekerja secara bersama-sama pada proyek yang sama. Mereka dapat bekerja pada cabang (branch) yang berbeda, menggabungkan perubahan, dan berbagi kode dengan mudah.

  3. Menciptakan versi (versioning):
    4. Git memungkinkan kalian membuat versi perangkat lunak pada titik tertentu dalam waktu. Ini memudahkan untuk mengembalikan kode ke versi sebelumnya jika terjadi masalah atau untuk merilis versi baru perangkat lunak.

C. Karakteristik Git

  1. Tersebar (Distributed):
    2. Setiap anggota tim memiliki salinan lengkap dari repositori Git, termasuk sejarah perubahan. Ini memungkinkan pengembang untuk bekerja secara independen di repositori lokal mereka.

  2. Cepat dan Efisien:
    3. Git dirancang untuk bekerja dengan cepat, bahkan pada proyek-proyek besar. Algoritma yang efisien memungkinkan Git mengelola riwayat perubahan dengan efisien.

    Cabang (Branching): Git mendukung penggunaan cabang yang memungkinkan pengembang untuk bekerja pada fitur atau perbaikan bug baru di cabang terpisah tanpa memengaruhi kode di cabang utama.

  3. Merge dan Rebase:
    4. Git menyediakan alat untuk menggabungkan perubahan dari cabang yang berbeda, baik dengan menggunakan penggabungan (merge) atau penggabungan ulang (rebase).

  4. Riwayat Perubahan (Commit History):
    5. Git mencatat setiap perubahan yang dibuat dalam kode sumber, termasuk pesan komit yang menjelaskan tujuan perubahan tersebut.

  5. Ketahanan Terhadap Kerusakan Data:
    6. Git dirancang untuk mengatasi kerusakan data dan kerusakan yang mungkin terjadi pada repositori.


GITHUB

A. Pengertian

  • Menurut Tom Preston-Werner (Salah satu pendiri GitHub):

    • GitHub adalah layanan hosting web untuk pengembangan perangkat lunak yang memungkinkan kolaborasi dengan menggunakan Git. Ini adalah tempat di mana pengembang dapat menyimpan kode sumber mereka, berkolaborasi dalam proyek, dan berbagi kode dengan dunia.

  • Menurut Situs Resmi GitHub:

    • GitHub adalah platform pengembangan perangkat lunak yang menggunakan Git sebagai sistem kontrol versi untuk mengelola dan menyimpan kode sumber. Ini adalah tempat di mana pengembang dapat berkolaborasi dalam proyek, mengelola riwayat perubahan kode, dan memungkinkan kolaborasi pengembangan perangkat lunak yang efisien.

B. Fungsi GitHub

  1. Manajemen Repositori:
    2. GitHub memungkinkan pengguna untuk membuat repositori Git (proyek perangkat lunak) dan mengelola riwayat perubahan dalam repositori tersebut.

  2. Kolaborasi Tim:
    3. Pengembang dapat bekerja secara bersama-sama dalam repositori yang sama. Mereka dapat berkontribusi ke proyek dengan mengirim permintaan tarik (pull requests) yang memungkinkan pemilik repositori untuk meninjau dan menggabungkan perubahan.

  3. Issue Tracking:
    4. GitHub menyediakan sistem pelacakan masalah (issue tracking) yang memungkinkan pengguna untuk melaporkan masalah, perbaikan bug, atau permintaan fitur dalam proyek.

  4. Wiki:
    5. GitHub menyediakan fitur wiki yang memungkinkan pengguna untuk membuat dan mengelola dokumentasi proyek secara terintegrasi.

  5. Papan Proyek (Project Boards):
    6. GitHub menyediakan papan proyek yang dapat digunakan untuk mengelola tugas dan mengatur alur kerja pengembangan proyek.

  6. GitHub Actions:
    7. Ini adalah fitur otomatisasi yang memungkinkan pengguna untuk membuat dan menjalankan alur kerja (workflows) otomatis, seperti pengujian, penerbitan, dan lainnya.

  7. Integrasi dengan Berbagai Layanan:
    8. GitHub dapat diintegrasikan dengan berbagai alat pengembangan, seperti Jenkins, Travis CI, Docker, dan banyak lagi.

C. Karakteristik GitHub

  1. Repositori Publik dan Pribadi:
    2. GitHub memungkinkan pengguna untuk membuat repositori publik yang dapat diakses oleh siapa saja atau repositori pribadi yang hanya dapat diakses oleh pengguna yang diundang.

  2. Kode Sumber Terbuka:
    3. Banyak proyek perangkat lunak sumber terbuka disimpan di GitHub, yang memungkinkan komunitas pengembang untuk berkontribusi secara terbuka.

  3. Layanan Hosting:
    4. GitHub menyimpan repositori Git di cloud, menghilangkan kebutuhan untuk pengelolaan server sendiri.

  4. Sistem Keamanan:
    5. GitHub menyediakan fitur keamanan yang kuat, termasuk otorisasi akses, otentikasi dua faktor, dan pemantauan keamanan kode.

  5. Pengelolaan Kolaborator:
    6. Pemilik repositori dapat mengelola siapa saja yang memiliki akses untuk berkontribusi ke repositori mereka.

  6. Statistik dan Analisis:
    7. GitHub menyediakan statistik tentang aktivitas repositori, kontributor, dan lainnya untuk membantu pengguna memahami perkembangan proyek mereka.

D. Terminologi GitHub

  1. Repository (Repo):
    2. Sama dengan dalam Git, ini adalah tempat penyimpanan utama untuk kode sumber dan sejarah perubahan, tetapi di-hosting di GitHub.

  2. Issues:
    3. Masalah atau tugas yang digunakan untuk melacak pekerjaan yang harus dilakukan dalam proyek.

  3. Wiki:
    4. Halaman-halaman dokumentasi yang dapat dibuat dan dikelola oleh pengguna untuk menjelaskan proyek.

  4. Project:
    5. Papan proyek yang dapat digunakan untuk mengelola dan mengorganisasi tugas-tugas proyek.

  5. Gist:
    6. Snippet kode sumber yang dapat dibagikan di GitHub.

  6. Organization:
    7. Akun khusus di GitHub yang dapat digunakan untuk mengelola repositori dan kolaborator.

  7. Contributors:
    8. Orang-orang yang berkontribusi pada repositori tertentu di GitHub.

  8. Star:
    9. Tindakan memberikan tanda bintang (star) pada repositori untuk menandai minat pada proyek tersebut.

  9. Watch:
    10. Tindakan mengikuti repositori untuk menerima pembaruan dan notifikasi terkait proyek.

  10. Fork:
    11. Sama dengan dalam Git, ini adalah tindakan membuat salinan repositori di GitHub untuk berkontribusi atau bekerja pada proyek.

  11. Pull Request (PR):
    12. Permintaan yang diajukan oleh kontributor untuk menggabungkan perubahan mereka ke repositori asal.


GIT BASH

Git Bash adalah sebuah terminal atau shell yang digunakan untuk menjalankan perintah-perintah Git dan perintah-perintah UNIX di sistem operasi Windows. Git Bash merupakan salah satu cara yang populer digunakan oleh pengembang perangkat lunak untuk berinteraksi dengan Git, yang merupakan sistem pengendalian versi yang sangat umum digunakan dalam pengembangan perangkat lunak.

Hubungan antara Git, Github, dan Git Bash:

  • Git adalah sistem pengendalian versi yang digunakan untuk mengelola perubahan kode sumber.

  • GitHub adalah platform hosting kode sumber yang memanfaatkan Git untuk menyediakan alat kolaborasi, manajemen repositori, dan alur kerja pengembangan yang efisien.

  • Git Bash adalah terminal yang digunakan di Windows untuk menjalankan perintah Git dan perintah berbasis UNIX yang diperlukan dalam pengembangan perangkat lunak yang menggunakan Git, terutama ketika bekerja dengan repositori di GitHub.

Tahapan membuat repository baru di akun GitHub
  1. Masuk ke akun GitHub yang telah dibuat.
  2. Klik "New" pada menu utama.
  3. Berikan nama pada repository baru kalian.
  4. Jika diperlukan, tambahkan deskripsi.
  5. Pilih opsi "Public" jika kalian ingin memungkinkan orang lain untuk mengakses repository kalian, atau pilih "Private" jika tidak ingin repository kalian dapat diakses oleh publik.

Perintah Dasar Git

  1. git init: Perintah ini digunakan untuk menginisialisasi repositori Git baru di direktori. Setelah menjalankan perintah ini, Git akan mulai melacak perubahan dalam direktori tersebut.

  2. git status: Perintah ini digunakan untuk melihat status saat ini. Ini memberi tahu tentang perubahan yang belum di-commit, cabang yang aktif, dan banyak informasi lainnya.

  3. git log: Perintah ini digunakan untuk melihat local komit dalam repositori. Ini menampilkan daftar semua komit yang telah dibuat beserta informasi seperti ID komit, penulis, tanggal, dan pesan komit.

  4. git add: Perintah ini digunakan untuk menambahkan perubahan yang ada dalam direktori kerja ke indeks (staging area). Kalian dapat menambahkan file atau direktori tertentu untuk disiapkan untuk commit.

  5. git commit: Perintah ini digunakan untuk membuat komit dari perubahan yang sudah di-staging. Kalian juga harus menyertakan pesan komit yang menjelaskan tujuan atau perubahan yang dilakukan.

  6. git push: Perintah ini digunakan untuk mengirimkan komit yang telah dibuat ke repositori jarak jauh (biasanya di server). Ini berguna ketika bekerja dengan repositori yang terpusat seperti yang terdapat di GitHub atau GitLab. Dengan push, perubahan kalian akan tersedia untuk kolaborator lainnya.

  7. git remote add origin: digunakan untuk menambahkan remote repository (repositori jarak jauh) ke repositori Git local. Remote repository adalah repositori yang berada di server atau tempat penyimpanan jarak jauh seperti GitHub, GitLab, Bitbucket, atau server Git lainnya. Dengan menambahkan remote repository, kalian dapat menghubungkan repositori Git local dengan repositori jarak jauh tersebut.


Git Branching

  1. git branch: Perintah ini digunakan untuk melihat daftar cabang (branch) yang ada dalam repositori Git. Ini juga dapat digunakan untuk membuat cabang baru atau beralih ke cabang yang ada.

  2. git fetch: Perintah ini digunakan untuk mengambil semua perubahan dari repositori remote (biasanya GitHub) dan menyimpannya di repositori lokal. Ini tidak akan menggabungkan perubahan ke cabang saat ini, itu hanya mengambil perubahan dan menyimpannya untuk dilihat.

  3. git checkout: Perintah ini digunakan untuk beralih antara cabang-cabang yang ada di repositori Git. Ini juga dapat digunakan untuk menciptakan cabang baru dan berpindah ke cabang tersebut.

  4. git clone: Perintah ini digunakan untuk menggandakan atau menyalin repositori Git yang ada di repositori remote (GitHub) ke repositori lokal. Ini menciptakan salinan lengkap dari repositori, termasuk semua cabang dan sejarahnya.

  5. git pull: Perintah ini digunakan untuk mengambil perubahan dari repositori remote (GitHub) dan menggabungkannya langsung ke cabang saat ini yang ada di repositori local.
